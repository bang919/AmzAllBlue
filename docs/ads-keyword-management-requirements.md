# 广告关键词管理需求

## 目标

在 AmzAllBlue 项目中新增一套广告关键词管理系统。

这套系统按照新的规则创建和管理广告，不整理、不导入，也不接管 Amazon 账户中原有的其他广告。

系统只管理以下广告组合中的广告：

```text
AmzAllBlue_ERP
```

不在 `AmzAllBlue_ERP` 广告组合中的广告，系统不读取、不修改、不暂停。

## 核心使用方式

前端使用表格管理关键词：

- 一个产品可以添加多个关键词。
- 一个关键词在主表中占一行。
- 关键词分为 `普通`、`主推`、`已稳定` 三组。
- 每个关键词可以分别开启或暂停 `精准`、`词组`、`广泛`。
- 点击关键词行后，展开查看每种投放方式的详细数据。

主表大致显示：

```text
父 ASIN | 子 ASIN | 关键词 | 分组 | 精准 | 词组 | 广泛 | 花费 | 销售额 | ACOS
```

展开后显示：

```text
广告类型 | 匹配方式 | 状态 | 出价 | 顶部加价 | 曝光 | 点击 | 花费 | 订单 | 销售额 | ACOS
```

## 广告创建位置

系统新建的广告统一创建在：

```text
广告 Profile：用户当前选择的 Profile，默认 US
广告组合：AmzAllBlue_ERP
```

第一版只创建和管理 SP 关键词广告。

每个关键词可以创建三套独立 Campaign：

```text
SP 精准：EXACT
SP 词组：PHRASE
SP 广泛：BROAD
```

三套投放相互独立，可以使用不同的出价、顶部加价和启停状态。

只有用户开启某种匹配方式时才创建对应广告。关闭时暂停广告，不删除，重新开启时继续使用原广告。

### 默认创建模板

首次创建使用以下默认设置：

```text
Campaign 日预算：8（使用当前 Profile 的货币单位，US 即 8 USD）
Campaign 竞价策略：固定竞价
默认勾选的匹配方式：EXACT
PHRASE 和 BROAD：默认不勾选，勾选后才创建
默认出价：0.2（使用当前 Profile 的货币单位）
顶部搜索结果加价：200%
开始日期：立即
初始状态：ENABLED
结束日期：不设置
```

顶部加价会叠加在基础出价上。例如基础出价为 `0.2`、顶部加价为 `200%` 时，顶部搜索结果位置的竞价最高可达到 `0.6`。

创建页面可以读取 Amazon Ads API 或 SIF 返回的建议竞价，作为推荐值展示或填入表单，用户确认后再创建。

系统按 Profile 分别保存上一次创建广告时使用的设置。下一次创建时自动套用为模板，所有值仍可修改；用户修改并成功创建后，更新该 Profile 的模板。

## 广告结构

第一版采用以下结构：

```text
Portfolio: AmzAllBlue_ERP
  Campaign: 一个关键词的一种匹配方式
    Ad Group: 一个子 ASIN 一个广告组
      Product Ad: 只投放该广告组对应的子 ASIN
      Keyword Target: 对应关键词和匹配方式
```

例如关键词 `wall organizer` 同时开启三种匹配方式，会创建三个 Campaign；如果需要投放多个子 ASIN，每个 Campaign 内为每个子 ASIN 分别创建 Ad Group：

```text
wall organizer / EXACT
wall organizer / PHRASE
wall organizer / BROAD
```

这样可以分别控制三种匹配方式的顶部加价和预算，也可以在 Keyword Target 层为不同子 ASIN 设置不同出价。

同一 Campaign 内的多个 Ad Group 共享 Campaign 日预算、竞价策略和顶部加价。如果将来需要让不同子 ASIN 使用不同的顶部加价，则需要为该子 ASIN 单独创建 Campaign。

## 命名规则

### Campaign 名称

格式：

```text
ERP__产品名__关键词__分组__SP__匹配方式
```

示例：

```text
ERP__WallBag__wall organizer__普通__SP__EXACT
ERP__WallBag__wall organizer__主推__SP__PHRASE
ERP__WallBag__wall organizer__已稳定__SP__BROAD
```

字段含义：

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 系统标识 | `ERP` | 表示由本系统管理 |
| 产品名 | `WallBag` | 产品的系统简称 |
| 关键词 | `wall organizer` | 实际投放关键词 |
| 分组 | `普通` | `普通`、`主推`、`已稳定` |
| 广告类型 | `SP` | 第一版固定为 SP |
| 匹配方式 | `EXACT` | `EXACT`、`PHRASE`、`BROAD` |

字段之间统一使用双下划线 `__` 分隔。

### Ad Group 名称

Ad Group 使用 Campaign 名称，并在末尾增加投放的子 ASIN：

```text
ERP__WallBag__wall organizer__主推__SP__EXACT__ASIN-B0B37GZ7MM
```

一个 Ad Group 只投放一个子 ASIN，因此 Ad Group 名称必须包含该子 ASIN。

### 分组改名

用户在前端将关键词从 `普通` 调整到 `主推` 时，系统同步修改对应 Campaign 和它下面所有 Ad Group 的名称。

修改前：

```text
ERP__WallBag__wall organizer__普通__SP__EXACT
```

修改后：

```text
ERP__WallBag__wall organizer__主推__SP__EXACT
```

只修改名称中的分组，不重新创建广告，不丢失历史数据。

### 名称长度和特殊字符

- 产品名使用项目中维护的产品简称。
- 关键词保留原文，方便在 Amazon 后台识别。
- 如果名称超过 Amazon 限制，系统自动缩短产品名或关键词显示部分。
- 系统在数据库保存 Amazon 返回的 Campaign ID、Ad Group ID 和 Target ID，名称主要用于查看和管理，不作为唯一识别依据。

## 分组规则

关键词有三个分组：

```text
普通
主推
已稳定
```

分组只表示运营管理状态，不自动决定广告是否开启。

用户可以随时修改分组，系统通过改名把新的分组同步到 Amazon Ads。

### 普通

目标：测试关键词价值，并通过较低成本获取有利润的订单。

策略：

- 使用相对保守的出价积累数据。
- 可以使用词组和广泛匹配发现有效搜索词。
- 使用精准匹配验证关键词本身的转化能力。
- 有稳定转化并值得提升自然排名的词，可以转为 `主推`。
- 流量不大但能够稳定盈利的长尾词，可以继续留在 `普通`。
- 长期只有点击而没有订单的词，降低出价或暂停。

重点指标：

```text
曝光
点击
CTR
CPC
订单
转化率
ACOS
广告利润
```

### 主推

目标：通过增加关键词广告成交，提升关键词自然排名，最终增加自然流量和自然订单。

策略：

- 以精准匹配作为主要投放方式。
- 根据表现提高精准出价和顶部搜索结果加价。
- 在可控范围内允许比利润型广告更高的 ACOS。
- 观察广告订单增长是否带来自然排名、自然订单和总订单增长。
- 同时关注 FBA 库存，避免排名提升后因断货失去排名。
- 自然排名和订单连续一段时间稳定后，转为 `已稳定`。

重点指标：

```text
关键词自然排名
广告订单
总订单
自然订单
自然订单占比
顶部曝光
ACOS
TACOS
库存可售天数
```

### 已稳定

目标：保持已有自然排名和销量，同时减少不必要的广告花费，提高整体利润。

策略：

- 保留必要的精准广告，维持关键词曝光和成交。
- 逐步测试降低出价或顶部搜索结果加价。
- 减少词组和广泛匹配中的无效流量。
- 观察降低广告投入后，自然排名和自然订单是否仍然稳定。
- 如果自然排名或自然订单明显下降，将关键词重新转为 `主推`。

重点指标：

```text
关键词自然排名稳定性
自然订单
自然订单占比
ACOS
TACOS
广告利润
总销量
```

### 分组流转

典型流转过程：

```text
普通 → 主推 → 已稳定
             ↓
          排名下降
             ↓
            主推
```

三个分组的核心定位：

```text
普通：测试关键词和低价盈利
主推：提升关键词自然排名
已稳定：保持排名并提高利润
```

## 数据记录

广告表现按天保存，方便查询任意日期范围、比较趋势和后续进行 AI 分析。

前端的一行仍然代表一个父 ASIN 下的一个关键词；数据库不要把产品配置和每天的表现数据全部重复写在一张表里。

### 关键词

每个关键词保存：

```text
父 ASIN
关键词
分组
创建时间
更新时间
```

### 子 ASIN

一个关键词可以推广一个或多个子 ASIN，一般只有一个。

每个子 ASIN 使用独立 Ad Group，不直接保存为数组。关联记录包括：

```text
投放单元 ID
子 ASIN
Seller SKU
Amazon Ad Group ID
Amazon Product Ad ID
状态
出价
```

这样可以为不同子 ASIN 单独设置出价，也可以单独增加、暂停或移除某个子 ASIN。

### Campaign 与投放单元

每种匹配方式保存一条 Campaign 记录：

```text
广告类型：SP / SB / SD
匹配方式：EXACT / PHRASE / BROAD
Campaign ID
状态：ENABLED / PAUSED
日预算
竞价策略
顶部加价
```

Campaign 下每个子 ASIN 保存一条 Ad Group 投放单元记录：

```text
Ad Group ID
Product Ad ID
Target ID
子 ASIN
状态：ENABLED / PAUSED
出价
```

第一版只使用 `SP` 和三种关键词匹配方式。

Campaign 代表一个关键词的一种匹配方式；Campaign 下的每个 Ad Group 投放单元代表一个被投放的子 ASIN。

### 每日表现

每个实际存在的子 ASIN Ad Group 投放单元每天最多保存一条：

```text
日期
Ad Group 投放单元 ID
曝光
点击次数
花费
订单数
购买产品数
销售额
数据同步时间
```

同一天重复同步时更新原记录，不重复新增。

以下指标查询时计算，不重复保存：

```text
CPC = 花费 / 点击次数
CTR = 点击次数 / 曝光
ACOS = 花费 / 销售额
ROAS = 销售额 / 花费
转化率 = 订单数 / 点击次数
```

没有创建、已经暂停且当天没有表现数据的投放单元，不需要插入全为零的每日记录。

如果一个关键词投放多个子 ASIN，每个子 ASIN Ad Group 分别保存每日表现；主表再将这些明细汇总为该关键词的数据。

### 同步规则

- 所有日期按当前 Amazon Ads Profile 的时区计算。
- 系统每小时尝试同步一次今天的数据；同一同步任务尚未结束时，不重复启动。
- 每天另外同步并覆盖最近 30 天的数据，用于补齐延迟归因的订单和销售额。
- 同步通过 Amazon Ads 报表异步完成，遇到 API 限流时排队重试，不并发重复申请相同报表。
- 页面日期弹窗参考 FBA 库存模块：已经成功请求并保存的日期显示绿点。
- 最近两天的数据仍可能变化，即使已经请求，也不显示绿点。
- 用户可以手动选择日期范围重新同步，结果按日期和投放单元覆盖，不重复新增。

### 自然排名

关键词自然排名后续通过 SIF API 或 MCP 获取，并按日期保存。

第一版广告数据同步不依赖 SIF；SIF 暂时不可用时，不影响 Amazon Ads 广告的创建、启停和表现数据同步。

### 创建失败处理

创建 Campaign、Ad Group、Product Ad 和 Keyword Target 时，每成功一步立即保存 Amazon 返回的 ID。

如果中途失败：

- 将该投放单元标记为 `创建未完成`，并显示失败步骤和原因。
- 已成功创建的对象不自动删除。
- 用户重试时从失败步骤继续。
- 重试前先检查已经保存的 Amazon ID，不重复创建已成功的对象。

### 数据量

按每个关键词投放一个子 ASIN，以 `100 个产品 × 100 个词 × 3 种 SP 匹配方式` 的满负荷估算：

```text
每天约 30,000 条
十年约 1.095 亿条
预计数据库和索引约 25～50 GB
```

Windows 数据盘建议为数据库、备份和运行空间预留约 `150 GB`。

第一版保留每日数据。数据积累多年后，可以保留最近两年的每日数据，并把更早的数据汇总为月度数据。

如果同一个关键词投放多个子 ASIN，每增加一个子 ASIN，会相应增加该关键词的 Ad Group 和每日数据量。

## 第一版功能

第一版只实现：

1. 识别并使用 `AmzAllBlue_ERP` 广告组合。
2. 选择产品并新增关键词。
3. 设置关键词为 `普通`、`主推` 或 `已稳定`。
4. 创建 SP 精准、词组、广泛广告。
5. 默认只勾选 EXACT，并使用可编辑、可记忆的创建模板。
6. 一个 Campaign 支持多个子 ASIN Ad Group，每个 Ad Group 只投放一个子 ASIN。
7. 分别开启或暂停三种匹配方式及其子 ASIN Ad Group。
8. 修改关键词分组并同步 Campaign 和 Ad Group 名称。
9. 表格显示曝光、点击、花费、订单、销售额和 ACOS。
10. 点击关键词查看三种匹配方式及各子 ASIN 的详细数据。
11. 每小时同步今天，每天滚动覆盖最近 30 天的广告表现。
12. 日期弹窗标记已请求数据，最近两天不显示完成绿点。
13. 创建中途失败后可以从失败步骤继续，不重复创建对象。

第一版暂不实现：

- SB 和 SD 广告。
- AI 自动调价。
- 自动管理 `AmzAllBlue_ERP` 之外的广告。
- 自动导入 Amazon 账户已有广告。

## 后续扩展

第一版稳定后再增加：

- AI 分析出价、曝光、转化和 ACOS。
- AI 提出调价或启停建议，由用户确认后执行。
- 调整历史和建议回顾。
- SB、SD 等其他广告类型。
- 搜索词分析和否定关键词建议。

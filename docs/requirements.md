# 亚马逊聚合器需求文档

## 产品定位

亚马逊聚合器是一个只在本机运行的运营工作台，用来聚合 Amazon 相关页面中的红人消息、产品 ASIN、活动数据、发布链接和后续动作。当前版本以 Creator Connections 红人合作为第一个完整模块，后续扩展到产品库、活动数据和更多采集器。

## 当前模块

### 总览

展示本地聚合数据的核心指标：

- 红人记录数量
- 涉及 ASIN 数量
- 可寄样数量
- 已发布数量
- 可复投数量
- 下一步动作队列

### 红人合作

保留并扩展原“红人样品合作管理”能力：

- 手动粘贴消息原文并解析
- Chrome 插件采集 Amazon 右下角消息
- 自动提取姓名、地址、ASIN、Storefront、YouTube、发布链接
- 自动分类为新申请、可寄样、等待视频、已发布、可复投等阶段
- 生成行动建议
- 生成英文邮件草稿
- 本地 AI 问答

### 产品库

下一阶段模块。目标是按 ASIN 聚合：

- 申请该产品的红人
- 已寄样红人
- 已发布视频链接
- 复投建议
- 后续可接产品页面、价格、评论、竞品等数据

### FBA 库存

用于监控美国站 FBA 库存、销量、发货和补货节奏。第一版按单日请求销量并记录每日库存/销量事实表，另用 SKU 商品资料表维护 ASIN、品名、图片等稳定信息。当天库存使用 FBA Inventory 当前快照；旧日期优先使用 FBA Ledger Summary 回填仓内可售/不可售，但旧日期没有入库计划、已发货、接收中、预留等细分，页面用 `/` 表示缺失，且旧日期总货量不展示为 0。7 天内允许刷新销量，超过 7 天的既有记录默认冻结；已保存过实时库存快照的日期，后续同步不覆盖库存字段，只更新销量字段。后续发补货建议按普通、主推、特推三档计算，日销量排除断货日，发货扣除 30 天运输期，补货扣除 30 天生产期 + 30 天运输期。

详细需求见 [FBA 库存功能需求](./fba-inventory-requirements.md)。

### 活动数据

下一阶段模块。目标是聚合 Creator Connections 活动数据：

- 活动名称
- 活动 ID
- 预算、花费、订单、销量
- 已接受创作者数量
- 已提交内容数量
- ASIN 级表现

### 采集器

统一管理 Chrome 扩展和备用脚本。当前已支持：

- Amazon Advertising 右下角消息采集
- 批量导入到本地 `/api/import/messages`
- 按会话名、邮箱或地址去重

## 本地存储

红人合作数据保存在 `data/db.json`。第一版继续使用 JSON 文件，便于备份和迁移。FBA 库存使用本机 MySQL 的 `fba_inventory_daily` 和 `fba_sku_metadata` 持久化，并通过 `.env` 中的 `DB_*` 配置连接。

## Chrome 插件采集器使用步骤

1. 启动本地系统：`npm start` 或 `node server.mjs`。
2. 打开 Chrome 扩展管理页：`chrome://extensions/`。
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择项目目录下的 `chrome-extension` 文件夹。
6. 打开 Amazon Advertising 内容创作者计划页面。
7. 进入活动详情，并点击右下角“消息”。
8. 页面右上角会出现“亚马逊聚合器采集器”面板。
9. 先点“采集 10”测试；确认正常后再点“采集 50”。
10. 回到本地系统点击“刷新”查看导入结果。

## 非目标

当前版本不做：

- 绕过 Amazon 登录或权限
- 自动发送消息
- 自动读取未展开或未授权的邮箱
- 云同步
- 多用户权限
- 自动下单或自动发货

## 下一步

建议优先做：

1. 产品库：按 ASIN 聚合红人、地址、视频和复投状态。
2. 活动数据采集：读取 Creator Connections 活动详情页中的活动表现。
3. 采集器配置页：显示插件安装状态、最后采集时间、导入数量。
4. 数据备份：一键导出 `db.json` 或 CSV。
